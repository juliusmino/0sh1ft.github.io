Manage OpenShift Container Platform

Use the web console to manage and configure an OpenShift cluster

To manage and configure an OpenShift cluster using the web console, follow these steps: Access the Web Console: Open your web browser and navigate to the URL provided during the OpenShift cluster installation1 . You should see a login page1 . Log In: Use the credentials provided during the installation (e.g., kubeadmin user and the password)1 . If you didn't install the cluster yourself, you can use the oc whoami --show-console command to find the web console URL1 . Navigate the Interface: Once logged in, you'll see two main perspectives: Administrator and Developer2 . The default perspective depends on your role2 . Administrator Perspective: This allows you to manage workload, storage, networking, and cluster settings2 . You can also install and manage Operators using the Operator Hub, add identity providers, and manage monitoring features2 . Developer Perspective: This perspective is focused on deploying applications, services, and databases2 . You can view real-time visualizations of rollouts, application status, and resource utilization2 . For more detailed instructions, you can refer to the OpenShift Container Platform documentation. https://docs.openshift.com/container-platform/4.10/web_console/web-console.html
